# MewCode Basic Chat Spec

## 背景
MewCode 是一个从零开始开发的命令行 AI 助手，目标形态类似 Claude Code，但本阶段只实现纯对话能力。当前项目是一个 Python 3.13 的空白 CLI 项目，尚未具备交互界面、模型调用、配置加载或多轮会话能力。

用户希望在终端启动 MewCode 后进入交互式对话界面，输入问题后由大模型 API 生成回复，并以流式方式逐字显示。系统需要支持 OpenAI 和 Anthropic Claude 两类后端，并通过项目级或用户级 YAML 配置文件切换当前使用的后端。

## 目标
- 提供一个可在终端启动的 MewCode 交互式对话入口。
- 支持用户连续输入多轮问题，并让 AI 回复能够参考本次会话中的历史上下文。
- 支持 OpenAI 和 Anthropic Claude 两种模型后端。
- 使用项目级或用户级 YAML 配置文件保存当前唯一生效的 LLM 供应商配置。
- 所有模型回复以流式方式输出，而不是等待完整回复生成后再显示。
- Provider 层保持统一抽象，便于后续新增模型后端。
- 支持配置 Claude extended thinking，但 TUI 只显示最终回复，不显示思考过程。

## 功能需求
- F1: 用户在终端启动 MewCode 后，系统进入交互式对话界面，显示可输入提示，并持续等待用户输入。
- F2: 用户输入一条问题后，系统将该问题发送给当前配置的模型后端，并把模型最终回复以流式文本形式显示在终端中。
- F3: 系统在同一次运行期间保留多轮对话历史，后续请求应包含此前用户输入和 AI 最终回复，使模型能够参考上下文。
- F4: 系统优先从当前运行目录下的项目级 YAML 配置文件读取当前唯一生效的 LLM 配置；项目级配置不存在时，回退到用户级 YAML 配置文件。配置字段包括供应商标识名、协议类型、模型名、请求地址、认证密钥，以及是否启用扩展思考。
- F5: 系统根据配置中的协议类型选择 OpenAI 或 Anthropic Claude 后端；如果协议类型不受支持，应给出明确错误并停止启动或停止本次请求。
- F6: 系统对 OpenAI 和 Anthropic Claude 暴露统一的 Provider 行为：接收当前会话消息与配置，流式产出可显示的最终回复文本片段。
- F7: 当配置选择 Anthropic Claude 且启用扩展思考时，系统应向后端请求扩展思考能力，但交互界面只展示最终回复文本，不展示思考内容。
- F8: 当配置文件缺失、配置字段缺失、认证失败、网络请求失败或模型服务返回错误时，系统应向用户显示可理解的错误信息，并保持程序状态可预期。
- F9: 用户可以通过明确的退出输入结束交互式会话。

## 非功能需求
- N1: 第一版使用 Python 3.13 开发，并保持项目结构适合后续扩展 agent 能力。
- N2: 流式输出应尽量低延迟，收到后端可展示文本片段后应立即写入终端。
- N3: Provider 抽象不应泄漏具体后端协议细节到交互界面层。
- N4: 配置文件中的认证密钥不应在正常输出、错误信息或日志中明文展示。
- N5: 单次会话历史只需保存在当前进程内，程序退出后无需持久化。
- N6: 第一版 TUI 以稳定、清晰、可用为优先，不要求复杂全屏布局；界面应接近 Claude Code 的简洁终端风格，并带有 MewCode 的猫猫文字画标识。
- N7: 错误处理应给出用户能采取行动的提示，例如检查配置文件、协议类型、网络、Provider 服务地址或认证密钥。

## 不做的事
- 不实现 tool use、函数调用、shell 命令执行或外部工具编排。
- 不读取、搜索、修改或生成用户项目文件。
- 不实现代码编辑、diff 展示、补丁应用或自动提交。
- 不实现项目上下文索引、代码库理解或长期记忆。
- 不实现对话历史持久化，程序退出后本次会话内容丢弃。
- 不实现多个配置 profile 或运行时配置切换命令。
- 不在 TUI 中展示 Claude extended thinking 的思考内容。
- 不实现复杂全屏 TUI、侧边栏、会话列表或富文本布局。
- 不提供图形界面、Web 服务或后台守护进程。

## 验收标准
- AC1: 启动 MewCode 后，终端进入可连续输入问题的交互式界面。
- AC2: 用户输入问题后，能看到模型回复以流式方式逐步显示，而不是一次性等待完整结果。
- AC3: 在同一次运行中连续提问时，后续回复能够参考前面用户与 AI 的对话内容。
- AC4: 当 `./.mewcode/config.yaml` 或 `~/.mewcode/config.yaml` 中配置 OpenAI 协议时，系统使用 OpenAI 后端完成一次流式对话。
- AC5: 当 `./.mewcode/config.yaml` 或 `~/.mewcode/config.yaml` 中配置 Anthropic Claude 协议时，系统使用 Claude 后端完成一次流式对话。
- AC6: 当项目级和用户级配置都存在时，系统使用项目级配置。
- AC7: 配置文件包含并校验六个字段：供应商标识名、协议类型、模型名、请求地址、认证密钥、是否启用扩展思考。
- AC8: Provider 层对交互界面提供统一行为，切换 OpenAI 与 Claude 不需要改变用户交互方式。
- AC9: Claude 配置启用扩展思考时，终端只显示最终回复，不显示思考内容。
- AC10: 配置缺失、协议不支持、认证失败、网络失败或服务错误时，终端显示清晰错误信息，且不泄露认证密钥。
- AC11: 用户输入明确退出命令后，交互式会话正常结束。
