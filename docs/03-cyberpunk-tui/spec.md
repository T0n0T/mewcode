# MewCode Cyberpunk TUI Spec

## 背景

MewCode 已支持交互式多轮对话、流式文本回复和工具调用，但当前终端界面仍采用逐行输入与直接打印的交互方式。

发送消息后，界面会先显示含义模糊的 `assistant` 标签，再等待首个流式片段。等待、接收回复、执行工具和生成最终答复等阶段缺少清晰区分。Markdown、代码、工具状态、错误和确认信息也都以普通文本连续输出，长会话中的阅读、回看和输入体验有限。

本里程碑重新设计终端交互界面，同时保持现有模型、Provider、工具和会话语义不变。

## 目标

- 建立接近 Codex CLI 的全屏、键盘优先终端界面。
- 使用克制的赛博朋克视觉语言形成 MewCode 自有品牌。
- 用明确、可计时的阶段状态替代 `assistant` 等模糊等待文字。
- 改善流式 Markdown、代码、工具事件、错误和确认信息的可读性。
- 提供常驻多行输入框、会话内输入历史和可靠的中断操作。
- 适配不同终端尺寸、颜色能力及非交互式输出环境。
- 严格限制为界面及必要展示状态改造，不扩展 Agent 能力。

## 功能需求

- **F1：全屏单列布局。** 交互式终端中使用全屏界面，由紧凑顶部栏、可滚动对话区和底部常驻输入区组成，不设置固定侧栏。
- **F2：启动与会话信息。** 空白会话显示小型猫咪品牌卡、MewCode 名称、当前模型、工作区及能力边界说明。顶部栏持续显示品牌、模型、工作区、Git 分支和连接状态。
- **F3：固定视觉主题。** 使用石墨黑背景、青蓝主色和洋红强调色；活动状态采用克制的旋转、呼吸和计时效果，静止内容不得闪烁或抖动。
- **F4：消息身份。** 用户消息使用 `›` 作为视觉锚点，MewCode 回复使用 `◆`；界面不得再显示 `assistant` 标签。
- **F5：明确的生成阶段。** 等待首个片段时显示 `UPLINKING <model> · <elapsed>`；工具运行时显示 `EXECUTING <tool>`；工具回灌后的最终生成阶段显示 `SYNTHESIZING · <elapsed>`。首个正文片段到达后，等待状态自然切换为回复内容。
- **F6：流式富文本。** 回复在接收过程中逐步渲染 Markdown，包括标题、列表、引用、表格、行内代码和代码块。未完成的结构不得造成明显闪烁或反复跳动。
- **F7：代码与宽内容。** 普通正文自动换行；代码、diff 和命令输出保持原始列结构，并允许横向查看超出区域的内容。
- **F8：智能滚动。** 用户位于底部时自动跟随流式输出；用户主动向上浏览后冻结当前位置，并显示有新内容的提示，支持快速返回底部。
- **F9：多行输入。** 输入框默认一行并按内容自动扩展，最多约六行；`Enter` 提交，`Shift+Enter` 换行，多行粘贴保持原始内容。
- **F10：草稿与输入历史。** 回复生成期间允许编辑下一条草稿，但不得提前提交、排队或插入当前请求。输入框为空时可浏览当前会话内已提交的提示词，退出后不持久化。
- **F11：中断与退出。** 生成期间按 `Esc` 或 `Ctrl+C` 立即中断；已显示内容保留并标记为 `INTERRUPTED`，但不加入后续模型上下文。输入框有内容时 `Ctrl+C` 清空内容；空输入框下再次按下可退出；`Ctrl+D` 在空输入框时退出，并继续兼容 `exit` 和 `quit`。
- **F12：工具事件。** 工具调用以对话内可展开事件卡显示脱敏后的关键参数、状态和耗时，大段结果默认折叠。需要确认的操作通过聚焦弹层展示命令或变更预览，并提供明确的批准与拒绝操作。
- **F13：错误反馈。** Provider、网络和工具错误在发生位置显示为对话内错误卡，包含简短原因及可执行建议；技术细节默认折叠，错误不得覆盖已有内容或直接终止会话。
- **F14：响应式布局。** 终端变窄时逐步隐藏 Git 分支等次要信息并简化装饰，优先保留对话、状态和输入区域；尺寸过小时显示明确提示，不得崩溃或产生错位输出。
- **F15：终端能力适配。** 优先使用 True Color，自动降级到 256 色或基础 ANSI 色；检测到 `NO_COLOR` 时移除颜色，但保留文字、符号和信息层级。
- **F16：非交互输出。** 输入或输出不是交互式终端时，自动切换为无动画、无全屏控制序列的线性纯文本模式，便于测试、日志记录和管道处理。
- **F17：界面语言。** 界面自身固定使用简洁英文状态与操作文案，不限制用户输入或模型正文使用的语言。

## 非功能需求

- **N1：行为兼容。** 除中断控制和展示状态所需的接口调整外，现有模型请求、Provider 协议、工具执行、安全确认、会话历史和配置查找语义必须保持不变。
- **N2：流式低延迟。** 界面不得为完整 Markdown 排版而等待整段回复；收到文本片段后应在下一个可见刷新周期内显示，状态计时持续更新。
- **N3：渲染稳定。** 流式更新、终端缩放、工具事件和弹层切换不得造成明显闪屏、内容重复、输入丢失或滚动位置无故跳动。
- **N4：长会话性能。** 随着对话和代码内容增长，输入、滚动及流式更新仍应保持响应，不得因每个片段反复重排全部历史而出现明显卡顿。
- **N5：信息可辨识。** 状态、成功、警告和错误不得只依赖颜色表达；在颜色降级或关闭时，仍可通过文字、符号和布局正确理解。
- **N6：密钥保护。** 顶部栏、工具事件、确认弹层、错误详情和纯文本回退输出均不得泄露 API key 或其他已登记的敏感值。
- **N7：中断一致性。** 用户中断后必须关闭当前流、停止继续渲染，并保证未完成回复不会作为完整助手消息进入后续上下文。
- **N8：输入可靠性。** 多行粘贴、快速键入、终端缩放和生成期间编辑草稿不得造成字符丢失、重复提交或意外排队。
- **N9：终端兼容。** 在常见现代终端中提供完整体验；不支持颜色、Unicode 或全屏控制能力时应可读地降级，而不是输出损坏的布局。
- **N10：可测试性。** 全屏交互、键盘操作、流式状态、终端缩放和非 TTY 回退均应能在不访问真实模型、不执行真实危险操作的条件下自动验证。
- **N11：范围克制。** 本里程碑不改变配置格式，不增加界面主题配置，也不借由界面改造引入尚未完成的 Agent 行为。

## 不做的事

- 不新增多步 Agent Loop、消息排队、运行中 steering 或其他 Agent 编排能力。
- 不新增斜杠命令、命令面板或快捷操作体系；仅保留现有退出行为。
- 不在界面中切换模型、Provider、权限策略或编辑配置。
- 不持久化对话、输入历史、草稿、滚动位置或界面状态。
- 不提供多主题、浅色主题、自定义配色或主题配置字段。
- 不加入固定侧栏、Token 遥测、上下文占用仪表盘或完整调试面板。
- 不改变现有工具集合、执行额度、安全边界或确认策略。
- 不允许生成期间提交、排队或插入下一条消息。
- 不默认展开读取、搜索、命令或其他工具产生的大段结果。
- 不实现操作系统剪贴板集成、桌面通知、声音提示或鼠标优先操作。
- 不实现中文界面、自动本地化或语言配置。
- 不开发桌面端、网页端或移动端界面。
- 不改变现有配置文件位置、字段及优先级。

## 验收标准

- **AC1（F1、F2）：** 在交互式终端启动 MewCode 后，可看到单列全屏界面，包括紧凑顶部栏、可滚动对话区、底部输入框，以及包含小型猫咪标记、模型和工作区信息的启动卡。
- **AC2（F2、F14）：** 顶部栏在空间允许时显示品牌、模型、工作区、Git 分支和连接状态；缩窄终端后次要字段依次隐藏，核心交互区域仍可使用。
- **AC3（F3、F15）：** 支持 True Color 时呈现石墨黑、青蓝和洋红主题；在 256 色、基础 ANSI 色及 `NO_COLOR` 环境中仍保持可辨识的信息层级。
- **AC4（F4）：** 用户消息以 `›` 标识，MewCode 回复以 `◆` 标识；启动、等待、流式输出、错误和纯文本回退中均不出现用户可见的 `assistant` 标签。
- **AC5（F5）：** 模拟首个片段延迟时，界面立即显示带模型名和递增耗时的 `UPLINKING`；首个片段到达后状态切换为正文，不遗留重复状态文本。
- **AC6（F5）：** 模型调用工具时可观察到 `EXECUTING <tool>`；工具结果回灌后的等待阶段显示 `SYNTHESIZING`，随后正确切换为最终正文。
- **AC7（F6、N2）：** 延迟返回多个 Markdown 片段时，用户可在流结束前看到逐步增长的内容；完成后标题、列表、引用、表格、行内代码和代码块均正确排版，过程中没有明显重复或闪烁。
- **AC8（F7）：** 普通长文本随窗口宽度换行；超宽代码、diff 和命令输出不破坏列结构，并可查看被隐藏的横向内容。
- **AC9（F8）：** 视图位于底部时自动跟随新内容；用户向上滚动后位置保持稳定并出现新输出提示；执行返回底部操作后恢复自动跟随。
- **AC10（F9）：** 输入框随内容从一行扩展到约六行；`Enter` 提交、`Shift+Enter` 换行，多行粘贴不会被拆成多次请求。
- **AC11（F10）：** 生成期间可以编辑下一条草稿，但提交操作不会排队或影响当前请求；当前回复结束后草稿仍然存在并可正常提交。
- **AC12（F10）：** 当前会话内可在空输入框中浏览已提交提示词；程序重启后不恢复这些记录。
- **AC13（F11、N7）：** 生成期间按 `Esc` 或 `Ctrl+C` 后不再显示后续片段，已有内容保留并标记 `INTERRUPTED`；下一次请求的模型上下文不包含该未完成回复。
- **AC14（F11）：** 输入框有内容时 `Ctrl+C` 清空输入；空输入框下再次按下退出；空输入框按 `Ctrl+D` 退出；输入 `exit` 或 `quit` 仍可结束会话。
- **AC15（F12）：** 工具运行期间显示可展开事件卡，其中包含脱敏后的工具名称、关键参数、状态和耗时；大段工具结果默认不展开。
- **AC16（F12、N6）：** 命令、写文件和改文件在执行前出现聚焦确认弹层，展示脱敏后的命令或变更预览；拒绝后不产生副作用，任何界面区域均不显示 API key。
- **AC17（F13）：** 模拟 Provider、网络或工具错误时，错误在对应位置显示原因和建议，技术细节默认折叠；关闭或离开错误卡后仍能继续输入下一条消息。
- **AC18（F14）：** 在宽、中、窄三种终端尺寸及运行中缩放的情况下，内容不重叠、草稿不丢失、滚动位置不无故重置；尺寸不足时显示可理解的提示而非异常退出。
- **AC19（F16）：** 将输入或输出重定向到非 TTY 环境时，程序输出线性纯文本记录，不包含全屏、光标移动、颜色或动画控制序列。
- **AC20（F17）：** 品牌、状态、确认、错误和退出等界面文案均使用英文；中文输入和中文模型回复可正常显示、换行与滚动。
- **AC21（N1、N11）：** 现有普通聊天、双 Provider、工具执行与确认、工作区安全、密钥脱敏、配置优先级和退出行为继续通过回归验证，配置格式保持不变。
- **AC22（N3、N4、N8）：** 在长对话、快速分片、生成期间输入草稿和终端连续缩放场景中，不出现重复提交、字符丢失、明显卡顿或无法恢复的界面状态。
- **AC23（N10）：** 全屏操作、状态转换、键盘行为、响应式布局和纯文本回退均能使用替代 Provider 与安全测试输入自动验证，无需真实网络或危险工具执行。
