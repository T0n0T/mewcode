# MewCode Tool System Spec

## 背景

MewCode 当前已具备交互式多轮对话、流式文本输出，以及 OpenAI 和 Anthropic 两种 Provider。现有运行时只能把用户消息交给模型并显示文本回复，无法执行模型提出的实际操作。

本阶段扩展此前“纯聊天”的边界，引入首个工具系统里程碑：模型可以根据用户请求选择一个工具，由 MewCode 在统一的安全与确认规则下执行，再将结构化结果回灌给模型生成最终回答。

## 目标

- 建立统一、可注册、可扩展的工具体系。
- 提供读文件、写文件、改文件、执行命令、按模式找文件和搜索代码内容六个核心工具。
- OpenAI 与 Anthropic 均支持流式工具调用识别、参数拼接、工具结果回灌和最终回答。
- 每轮最多执行一个工具；结果回灌后只允许模型生成最终文本，不进入自动 Agent Loop。
- 文件操作限制在启动工作区内，并阻止路径及符号链接逃逸。
- 命令、写文件和改文件必须经过用户确认；读取与搜索可以直接执行。
- 工具失败、拒绝、超时和输出截断均以结构化结果反馈给模型，使其能够给出准确说明。
- 为下一阶段的多步 Agent Loop 保留清晰扩展点，但本阶段不提前实现循环编排。

## 功能需求

- **F1：统一工具定义。** 每个工具必须提供唯一名称、用途描述、JSON 参数 Schema 和可执行行为，并统一返回可序列化的结构化结果。
- **F2：工具注册中心。** 系统集中登记可用工具，支持按名称查找、拒绝重复名称，并能把全部工具元信息转换为当前模型 Provider 可识别的工具列表。
- **F3：读取文件。** 模型可以读取工作区内已知路径的 UTF-8 文本文件。即使文件被 `.gitignore` 忽略，只要路径明确且未逃逸工作区，仍允许读取。
- **F4：写入文件。** 模型可以新建或完整覆盖工作区内的 UTF-8 文本文件；父目录缺失时自动创建。执行前必须展示目标路径和变更预览，并取得用户确认。
- **F5：修改文件。** 模型通过“原文到新文”的精确替换修改 UTF-8 文本文件。原文必须在文件中恰好出现一次；未匹配或匹配多次时不得修改，并返回可供模型重试的明确错误。执行前必须展示目标路径和差异预览，并取得用户确认。
- **F6：执行命令。** 模型可以提交一段完整 shell 命令，支持管道、重定向和条件连接。命令以 MewCode 启动目录为工作目录，执行前原样展示并取得用户确认；结果包含退出状态、标准输出和标准错误。
- **F7：按模式查找文件。** 模型可以在工作区内按路径模式查找文件。查找默认遵循 `.gitignore`，并始终排除 `.git` 目录。
- **F8：搜索代码内容。** 模型可以在工作区文本文件中搜索内容模式，结果应包含足以定位匹配项的文件路径、行号和匹配内容。搜索默认遵循 `.gitignore`，始终排除 `.git`，并跳过二进制或非 UTF-8 文件。
- **F9：工作区边界。** 所有文件类工具只能访问启动目录及其子目录。绝对路径、父目录跳转和符号链接均不得造成工作区逃逸。命令以工作区为默认目录，但本阶段不承诺操作系统级沙箱隔离。
- **F10：确认与拒绝。** 命令、写文件和改文件必须逐次确认。用户拒绝后不得产生副作用，并将“用户拒绝”作为结构化工具结果回灌给模型。读取和搜索不需要确认。
- **F11：超时与失败结果。** 普通工具固定在 30 秒后超时；命令默认超时 30 秒，并允许模型指定不超过 300 秒的正数超时。拒绝、超时、参数错误、路径越界、编码错误及执行异常均转换为结构化结果，不得使会话崩溃。
- **F12：输出限制。** 读取、查找、搜索和命令输出受到默认大小限制。超过限制时返回截断后的内容，并明确标记已截断、原始总量及建议模型缩小范围的信息。
- **F13：双 Provider 工具调用。** OpenAI 与 Anthropic 都必须接收工具定义，并把各自的流式响应统一转换为文本片段、工具调用标识、工具名称和 JSON 参数片段。系统应按调用标识正确拼接参数；参数不完整或无法解析时返回清晰错误。
- **F14：单工具回灌流程。** 模型首次响应没有工具调用时，系统沿用普通流式对话；包含一个有效工具调用时，系统执行该工具，将调用及结构化结果加入本轮上下文，再请求模型生成最终文本回答。
- **F15：单轮调用限制。** 首次响应同时包含多个工具调用时，一个都不执行，并把“每轮只允许一个工具调用”的结构化错误回灌给模型。工具结果回灌后，模型只允许输出最终文本；若再次请求工具，系统不执行并显示“本轮工具额度已用完”，随后结束该轮，不再发起第三次模型请求。
- **F16：终端可见性。** 工具运行时显示工具名称、关键参数和执行状态。读取、查找和搜索的完整结果只提供给模型，不直接刷满终端；命令、写入和修改额外显示用户确认所需的信息。

## 非功能需求

- **N1：安全边界。** 工作区路径校验必须覆盖规范化路径、父目录跳转、绝对路径和符号链接逃逸。所有副作用操作必须在确认后执行，预览阶段不得提前写盘或运行命令。
- **N2：密钥保护。** 工具参数、执行结果、异常、终端状态和 Provider 错误均不得泄露配置中的 API key。
- **N3：协议隔离。** OpenAI 与 Anthropic 的流式事件格式、工具定义格式和结果回灌格式只能存在于各自 Provider 层；运行时和工具层只依赖统一事件与消息语义。
- **N4：结果稳定性。** 所有工具结果采用一致的成功、失败、拒绝、超时和截断表达，使模型无需根据工具名称猜测错误格式。
- **N5：低延迟流式处理。** 普通文本回复继续即时显示；工具调用参数允许随流增量收集，不要求等待或缓存与本次调用无关的完整响应内容。
- **N6：会话一致性。** 只有协议完整、可用于后续请求的助手消息和工具结果才能加入历史。解析失败、多个工具调用或第二次工具请求不得留下无法被对应 Provider 重放的残缺历史。
- **N7：可测试性。** 工具执行、确认输入、时间控制、命令进程和 Provider 流均应可替换或注入，以便在测试中验证行为而不访问真实网络、不执行真实危险命令，也不修改工作区外内容。
- **N8：跨平台边界。** 文件工具使用平台无关的路径处理；命令工具使用当前运行环境的 shell 语义，不承诺同一条 shell 命令能跨操作系统运行。
- **N9：兼容性。** 不调用工具的现有 OpenAI、Anthropic 流式对话、配置查找顺序、退出命令和终端体验应保持可用。
- **N10：可扩展性。** 新增工具不应要求修改运行时主流程或现有工具实现；后续增加 Agent Loop 时应能复用注册、执行、确认、结构化结果和 Provider 事件机制。
- **N11：清晰反馈。** 确认提示必须清楚展示将发生的操作；错误信息应说明失败原因和可采取的下一步，同时避免输出内部堆栈或无关实现细节。
- **N12：实现约束。** 本阶段继续使用 Python 3.13 和现有 CLI 架构，改动应贴合现有配置、Provider、运行时和 TUI 模块边界。

## 不做的事

- 不实现可连续自主调用多个工具的 Agent Loop。
- 不执行模型一次响应中提出的多个并列工具调用。
- 不允许工具结果回灌后的第二次模型响应继续执行工具。
- 不实现并行工具调用、工具调用队列、任务规划器或自动重试编排。
- 不提供操作系统级沙箱、容器隔离、权限降级或网络隔离。
- 不对 shell 命令进行高低风险分类，也不自动批准任何命令。
- 不允许模型绕过确认策略；命令、写文件和改文件每次都必须确认。
- 不实现增量补丁、模糊匹配、正则替换或自动冲突解决；改文件只支持原文唯一匹配替换。
- 不处理二进制文件、非 UTF-8 文本、Base64 文件传输或文件编码转换。
- 不实现文件删除、移动、复制、目录删除或权限修改工具。
- 不实现 Git 专用工具、提交、分支管理或远程仓库操作。
- 不实现网络请求、网页浏览、外部搜索或第三方工具插件。
- 不实现工具权限配置、工具启停配置、项目级工具白名单或自定义超时配置。
- 不把完整工具结果默认打印到终端，也不新增复杂全屏工具面板。
- 不持久化工具调用历史；程序退出后仍丢弃本次会话。
- 不改变现有 LLM Provider 配置文件位置、字段和选择规则。
- 不承诺命令在不同操作系统或不同 shell 之间具有相同行为。

## 验收标准

- **AC1（F1、F2）：** 六个核心工具均可通过统一注册中心按名称找到；重复名称会被拒绝；系统可为 OpenAI 和 Anthropic 生成包含名称、描述及参数 Schema 的工具列表。
- **AC2（F3）：** 读取工作区内已知 UTF-8 文件时返回文件内容；明确读取被 `.gitignore` 忽略的文件仍可成功；文件不存在、不是普通文件或编码无效时返回结构化错误。
- **AC3（F4）：** 写文件能够新建文件、自动创建父目录及完整覆盖已有文件；执行前展示路径和变更预览，只有确认后才落盘。
- **AC4（F5）：** 改文件仅在原文恰好匹配一次且用户确认后完成替换；匹配零次或多次时文件保持不变，并返回说明原因的结构化错误。
- **AC5（F6）：** 命令确认后以启动目录为工作目录执行完整 shell 命令，并返回退出状态、标准输出和标准错误；拒绝确认时命令完全不执行。
- **AC6（F7）：** 按模式找文件能够返回工作区内匹配路径，默认不返回 `.git` 或 `.gitignore` 排除的内容。
- **AC7（F8）：** 搜代码内容能够返回匹配文件、行号和内容，默认排除 `.git`、被忽略文件、二进制文件和非 UTF-8 文件。
- **AC8（F9）：** 文件工具面对绝对路径、`..` 跳转或指向工作区外部的符号链接时均拒绝访问；工作区外文件不会被读取或改变。
- **AC9（F10）：** 每次命令、写入和修改都单独请求确认；拒绝后无副作用，且模型收到明确的结构化拒绝结果。读取、找文件和搜内容无需确认即可运行。
- **AC10（F11）：** 普通工具超过 30 秒会停止并返回超时结果；命令默认使用 30 秒，接受不超过 300 秒的有效超时，拒绝零值、负值或超过上限的设置。
- **AC11（F11）：** 参数错误、未知工具、路径越界、编码错误、用户拒绝、超时及执行异常均作为结构化结果交给模型，会话不会因此显示内部堆栈或异常退出。
- **AC12（F12）：** 读取、查找、搜索或命令产生超限结果时，只回灌限定大小的内容，同时明确指出结果已截断、原始总量以及缩小查询范围的建议。
- **AC13（F13）：** OpenAI 流中被拆分为多个片段的工具名称、调用标识和 JSON 参数能够被正确重组；Anthropic 的对应流式工具调用也能得到相同语义的统一结果。
- **AC14（F13）：** 流结束时参数仍不完整或不是合法 JSON，工具不会执行，模型会收到可理解的结构化解析错误。
- **AC15（F14）：** 模型未调用工具时，回复继续以现有方式流式显示；模型提出一个有效工具调用时，系统执行一次，将调用和结果回灌，再显示模型生成的最终文本回答。
- **AC16（F15）：** 首次响应包含多个工具调用时，任何工具都不会执行；模型收到“每轮只允许一个工具调用”的结果，并有机会生成最终文本说明。
- **AC17（F15）：** 工具结果回灌后，模型若再次请求工具，该工具不会执行；终端显示“本轮工具额度已用完”并结束本轮，不发起第三次模型请求。
- **AC18（F16）：** 工具运行期间终端显示工具名称、关键参数和最终状态；读取与搜索的完整结果不会直接刷到终端；命令、写入和修改会展示足够的确认信息。
- **AC19（N2、N9）：** 工具参数、结果、错误和终端输出均不包含 API key；现有 OpenAI 与 Anthropic 普通流式聊天、配置优先级、会话历史和退出行为继续通过回归验证。
